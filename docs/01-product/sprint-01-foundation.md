# LifeOS - Sprint 1 Design Documentation

## 1. Vision

## Mission

**Collect experiences. Produce insights.**

LifeOS is a personal growth operating system designed to turn everyday progress into meaningful data.

The goal is not simply to track tasks. The goal is to understand:

* What activities create progress?
* What motivates consistent action?
* Where are opportunities for improvement?
* How can decisions be improved through data?

LifeOS combines gamification, personal development, and analytics into one evolving system.

---

# 2. Core Principles

## Principle 1 - Progress should feel visible

Small actions should contribute toward larger goals.

A 20-minute learning session should not disappear. It should become measurable progress.

---

## Principle 2 - Low friction beats perfect tracking

The system should never become another responsibility.

Logging progress should take less than 30 seconds.

If tracking becomes harder than the activity itself, the system has failed.

---

## Principle 3 - Data creates insights

The purpose of collecting data is not collecting data.

The purpose is learning from it.

Examples:

* Which activities create the most motivation?
* Which projects are abandoned?
* What environments create the best performance?
* Which habits correlate with success?

---

# 3. MVP Game Structure

The MVP contains six core objects.

## Areas

Large categories of life.

Examples:

* Career
* Wealth
* Lifestyle
* Mind
* Health
* Relationships

Areas provide direction.

---

## Projects

A project is a meaningful objective with a beginning and an end.

Examples:

Career:

* Become a Data Analyst

Wealth:

* Develop Automated Trading Model

Lifestyle:

* Become a Better Cook

Mind:

* Improve Reflection Practice

Project statuses:

* Active
* Paused
* Completed
* Archived

Nothing is deleted. Completed projects become historical data.

---

## Quests

Quests are the smallest unit of progress.

A quest should normally take between 15 and 60 minutes.

Examples:

* Complete SQL exercise
* Analyze dataset
* Backtest trading idea
* Cook a new recipe
* Write reflection notes

A project always has a current next quest.

---

## XP System

XP measures progress.

Example:

Easy quest:
+10 XP

Medium quest:
+25 XP

Hard quest:
+50 XP

Major milestone:
+100 XP

XP contributes to:

* Levels
* Achievements
* Statistics
* Insights

---

# 4. Achievement System

Achievements are automatically unlocked milestones.

Examples:

Career:

* First Dataset Completed
* First Portfolio Project
* 100 Hours Learning

Trading:

* 50 Backtests Completed
* First Automated Strategy

Lifestyle:

* First Dinner Hosted
* 10 New Recipes Learned

Mind:

* 30 Reflection Entries

Achievements create recognition of progress.

---

# 5. Reward System

Rewards are optional external motivators tied to meaningful achievements.

Rules:

* Rewards should celebrate milestones.
* Rewards should not replace intrinsic motivation.
* Rewards should be defined before achievement completion.

Examples:

Small:

* Coffee
* Favorite snack
* Cheat meal
* Gift under €10

Medium:

* New book
* Cooking equipment
* Hobby item

Large:

* Experience
* Trip
* Bigger purchase

---

# 6. Core Game Loop

The LifeOS loop:

```
Choose Quest

↓

Complete Activity

↓

Earn XP

↓

Update Progress

↓

Unlock Achievements

↓

Generate Data

↓

Analyze Insights

↓

Improve Decisions
```

---

# 7. Daily User Experience

Opening LifeOS should answer three questions:

## What should I do?

Show:

* Current quests
* Recommended next actions

---

## How am I progressing?

Show:

* XP
* Levels
* Active projects
* Achievement progress

---

## What have I learned?

Show:

* Personal insights
* Trends
* Areas needing attention

---

# 8. MVP Success Criteria

The MVP succeeds if:

1. Logging progress feels effortless.
2. Progress feels rewarding.
3. Data is generated consistently.
4. Insights can be extracted from the data.
5. The system encourages long-term engagement.

---

Potential future features:
