# Decision 006 – Adaptive XP

## Status

Accepted (Future Evolution)

---

## Decision

LifeOS should eventually adapt XP to reflect personal growth rather than objective task difficulty alone.

Growth is relative to the individual.

The same activity may represent different levels of progress for different users.

---

## Problem

Fixed XP systems cannot account for changing skill levels or personal comfort zones.

They eventually reward repetition instead of growth.

---

## Rationale

LifeOS should learn from historical data.

As the dataset grows, the system should become better at estimating challenge and growth for each individual.

The MVP will rely on user-selected values.

Adaptive XP is a long-term objective.

---

## Trade-offs

Pros

* Highly personalized.
* Encourages continuous growth.
* Creates richer analytics.

Cons

* Requires significant historical data.
* More complex implementation.

---

## Consequences

The MVP should collect sufficient data to support future adaptive XP models.

Automatic adjustment should only occur after the system has learned meaningful user patterns.
