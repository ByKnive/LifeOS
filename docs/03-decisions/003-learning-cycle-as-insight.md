# Decision 003 – Learning Cycles Are Insights, Not Requirements

## Status

Accepted

---

## Decision

The learning cycle (Learn → Practice → Build → Experience → Reflect) will be used as an analytical framework rather than a mandatory progression path.

Users should never be required to complete every stage.

---

## Problem

Learning does not always follow a fixed sequence.

Forcing every project through predefined stages could create unnecessary pressure and reduce flexibility.

---

## Rationale

LifeOS should identify patterns rather than prescribe behavior.

The learning cycle helps users understand where they invest their effort without suggesting that one sequence is always correct.

---

## Trade-offs

Pros

* Preserves user autonomy.
* Enables meaningful analytics.
* Supports different learning styles.

Cons

* Less explicit guidance.
* Requires stronger insight generation.

---

## Consequences

Future dashboards may visualize learning patterns without treating missing stages as unfinished work.

Insights should encourage curiosity rather than completion.
